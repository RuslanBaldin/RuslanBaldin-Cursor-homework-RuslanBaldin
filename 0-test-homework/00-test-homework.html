Інструменти розробки (VSCode, Git, Github)

Перелік основних команд:

  • git config --global user.name blabla
  • git config --global user.email blabal@bla
  • git init - створює новий репозиторій Git. З її допомогою можна перетворити існуючий проект без керування версіями на репозиторій Git або ініціалізувати новий порожній репозиторій.
  • git status - відображає стан робочого каталогу та розділу проіндексованих файлів. З її допомогою можна перевірити індексацію змін та побачити файли, які не відстежуються Git. Інформація про історію комітів проекту не відображається під час виведення даних про стан. Для цього використовується команда git log.
  • git add - це перша команда в ланцюжку операцій, що наказує Git "зберегти" знімок поточного стану проекту в історії комітів. Коли git add використовується як окрема команда, вона переносить очікувані зміни з робочого каталогу розділ проіндексованих файлів.
  • git commit - це команда для запису індексованих змін до репозиторію Git. Перш ніж створювати черговий коміт, необхідно проіндексувати файли у робочій області за допомогою команди git-add.
  • git log - по дефолту (без аргументів) git log перераховує комміти, зроблені в репозиторії у зворотному до хронологічного порядку - останні коміти знаходяться вгорі.
  • git checkout - дозволяє переміщатися між гілками, створеними командою git branch. При перемиканні гілки відбувається оновлення файлів у робочому каталозі відповідно до версії, що зберігається в цій гілці, а Git починає записувати нові комміти в цій гілці.
  • git branch - це свого роду "менеджер гілок". Ця команда вміє перераховувати ваші гілки, створювати нові, видаляти та перейменовувати їх.
  • git push - команда дозволяє передати коміти з локального репозиторію у віддалений.
  • git pull - команда відповідає за завантаження даних з сервера. Процес дуже схожий на клонування репозиторію, але тут завантажуються не всі комміти, а лише нові.
  • git merge - виконує злиття окремих напрямів розробки, створених за допомогою команди git branch в єдину гілку.
  • git clone - це утиліта командного рядка Git для вибору існуючого репозиторію та створення його клону, тобто копії.

.gitignore файл - це, як правило, артефакти складання та файли, що генеруються машиною з вихідних файлів у вашому репозиторії, або файли, які з будь-якої іншої причини не повинні потрапляти до коммітів.

Advanced

Такi команди:

  • git revert - використовується для запису деяких нових комітів, щоб скасувати дію деяких попередніх комітів (можливо, несправних).
  • git reset - це складний універсальний інструмент для скасування змін. Команда має три основні форми виклику, що відповідають аргументам командного рядка --soft, --mixed, --hard.
  • git rebase - це процес переміщення послідовності комітів до нового базового коміту або їхнього об'єднання.
  • git cherry-pick - бере зміни, що вносяться одним комітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці.
  • git stash - бере змінений стан вашого робочого каталогу, тобто змінені файли, що відстежуються, і проіндексовані зміни, і зберігає їх у сховищі незавершених змін, які ви можете в будь-який час застосувати назад.
